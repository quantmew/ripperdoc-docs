Ripperdoc 提供 CLI 选项用于启动配置，以及 slash 命令用于运行时会话管理。

## CLI 选项

### 基本用法

```bash
ripperdoc [选项] [命令]
```

该命令也有别名 `rd`：

```bash
rd [选项]
```

### 启动选项

| 选项 | 说明 |
|--------|-------------|
| `--version` | 显示版本信息 |
| `--help` | 显示帮助信息 |
| `--cwd PATH` | 设置工作目录 |
| `--yolo` | 跳过所有权限提示 |
| `--verbose` | 启用详细输出 |
| `--model NAME` | 使用指定模型配置 |
| `-c`, `--continue` | 继续最近一次会话 |

### 提示选项

| 选项 | 说明 |
|--------|-------------|
| `-p`, `--prompt TEXT` | 直接运行提示词（非交互模式） |
| `--system-prompt TEXT` | 替换默认系统提示词 |
| `--append-system-prompt TEXT` | 追加到默认系统提示词 |

### 工具选项

| 选项 | 说明 |
|--------|-------------|
| `--tools LIST` | 指定可用工具（逗号分隔） |

工具值：
- `""` - 禁用所有工具
- `"default"` - 使用所有默认工具
- `"Bash,Edit,Read"` - 指定工具列表

### 显示选项

| 选项 | 说明 |
|--------|-------------|
| `--show-full-thinking` | 显示完整推理内容 |
| `--hide-full-thinking` | 显示截断的推理预览 |

### 示例

```bash
# 启动交互会话
ripperdoc

# 继续上次会话
ripperdoc --continue
rd -c

# 非交互式单次提示
ripperdoc -p "解释这个代码库"

# 自定义系统提示词
ripperdoc --append-system-prompt "专注于安全性"

# 限制工具
ripperdoc --tools "Read,Glob,Grep"

# YOLO 模式（无权限提示）
ripperdoc --yolo
```

## Slash 命令

### 通用命令

#### /help

显示可用命令与用法信息。

```
> /help
> /help tools
```

#### /exit

退出当前会话。

```
> /exit
```

#### /clear

清除对话历史并重新开始。

```
> /clear
```

### 会话命令

#### /status

显示当前会话状态，包括消息数、token 与时长。

```
> /status
```

#### /stats

显示更详细的会话统计。

```
> /stats
```

#### /cost

显示 token 使用与预估费用。

```
> /cost
```

#### /compact

手动压缩对话以减少上下文体积。

```
> /compact
> /compact 重点关注 API 细节
```

#### /resume

交互式选择并恢复之前的会话。

```
> /resume
```

### 配置命令

#### /config

查看或修改配置。

```
> /config
> /config set model.timeout 180
```

#### /models

列出并切换模型配置。

```
> /models
> /models use fast
```

#### /themes

列出并切换 UI 主题。

```
> /themes              # 显示当前主题并列出可用主题
> /themes light        # 切换到浅色主题
> /themes preview nord # 预览主题的调色板
```

可用主题：`dark`、`light`、`monokai`、`dracula`、`solarized_dark`、`nord`

### 工具命令

#### /tools

列出可用工具。

```
> /tools
```

#### /permissions

管理权限规则。

```
> /permissions
> /permissions list
> /permissions add allow Bash "npm test"
```

### 扩展命令

#### /hooks

管理 hooks 配置。

```
> /hooks
> /hooks list
> /hooks reload
```

#### /skills

列出可用技能。

```
> /skills
```

#### /skill

加载技能。

```
> /skill pdf
> /skill pptx
```

#### /agents

列出代理定义。

```
> /agents
```

#### /tasks

列出正在运行的子代理任务。

```
> /tasks
```

#### /mcp

管理 MCP 服务器连接。

```
> /mcp
```

### 任务管理

#### /todos

查看待办列表。

```
> /todos
```

#### /todo

添加或管理待办项。

```
> /todo add Implement feature X
> /todo done 1
```

### 记忆命令

#### /memory

查看并管理对话记忆。

```
> /memory
```

#### /context

查看当前上下文信息。

```
> /context
```

### 诊断

#### /doctor

运行诊断检查配置与连通性。

```
> /doctor
```

## 自定义命令

在 `.ripperdoc/commands.json` 中定义自定义命令：

```json
{
  "commands": {
    "test": "Run npm test and report results",
    "deploy": "Deploy to staging environment",
    "review": "Review code in $1 for issues"
  }
}
```

使用自定义命令：

```
> /test
> /review src/api.py
```

## 命令快捷方式

部分命令有快捷方式：

| 命令 | 快捷方式 |
|---------|----------|
| `/help` | `/h` |
| `/exit` | `/q` |
| `/clear` | `/c` |
| `/status` | `/s` |

## 获取帮助

查看某个命令的帮助：

```
> /help <command>
```

例如：

```
> /help permissions
```
